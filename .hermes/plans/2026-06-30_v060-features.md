# 8Router v0.6.0 — Tunnel + OAuth + i18n Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Add three major features to 8Router — secure tunnel sharing, OAuth authentication, and multi-language support (EN/ID/JA).

**Architecture:** Each feature is implemented as an isolated module with clean integration points. Tunnel uses a local proxy pattern (similar to ngrok). OAuth wraps the existing dashboard auth. i18n uses a JSON string map approach (no heavy framework).

**Tech Stack:** Express middleware, JWT tokens, JSON string catalogs, simple-fetch tunnel client.

---

## Feature 1: Tunnel (Secure Remote Access)

**Why:** Let users share their local 8Router instance with team members securely via a public URL, without manual ngrok/cloudflare tunnel setup.

**Approach:** Built-in tunnel that creates a secure HTTPS proxy from a public endpoint to the local 8Router. Uses a simple token-based auth for the tunnel connection. Dashboard remains local-only (tunnel only exposes API endpoints).

### Task 1.2: Create tunnel token middleware

**Objective:** Validate tunnel tokens on incoming requests

**Files:**
- Create: `src/tunnel/auth.ts`

```typescript
// src/tunnel/auth.ts
import { Request, Response, NextFunction } from 'express';

export function tunnelAuthMiddleware(validToken: string) {
  return (req: Request, res: Response, next: NextFunction) => {
    // Skip auth for local requests
    const isLocal = req.ip === '127.0.0.1' || req.ip === '::1' || req.ip === '::ffff:127.0.0.1';
    if (isLocal) return next();

    // Check tunnel token
    const token = req.headers['x-8router-token'] as string
      || req.query.token as string
      || extractBearerToken(req);

    if (!token || token !== validToken) {
      return res.status(401).json({
        error: { message: 'Invalid or missing tunnel token', code: 401 }
      });
    }

    next();
  };
}

function extractBearerToken(req: Request): string | null {
  const auth = req.headers.authorization;
  if (auth?.startsWith('Bearer ')) return auth.slice(7);
  return null;
}
```

### Task 2.2: Create JWT token utilities

**Objective:** Sign and verify JWT tokens for OAuth sessions

**Files:**
- Create: `src/auth/jwt.ts`

```typescript
// src/auth/jwt.ts
import crypto from 'crypto';

export interface JWTPayload {
  sub: string;      // user ID
  email: string;
  name: string;
  provider: string;
  iat: number;
  exp: number;
}

export function signJWT(payload: Omit<JWTPayload, 'iat' | 'exp'>, secret: string, expiresInHours: number = 24): string {
  const header = { alg: 'HS256', typ: 'JWT' };
  const now = Math.floor(Date.now() / 1000);
  const fullPayload = { ...payload, iat: now, exp: now + (expiresInHours * 3600) };

  const headerB64 = Buffer.from(JSON.stringify(header)).toString('base64url');
  const payloadB64 = Buffer.from(JSON.stringify(fullPayload)).toString('base64url');
  const signature = crypto.createHmac('sha256', secret)
    .update(`${headerB64}.${payloadB64}`)
    .digest('base64url');

  return `${headerB64}.${payloadB64}.${signature}`;
}

export function verifyJWT(token: string, secret: string): JWTPayload | null {
  try {
    const [headerB64, payloadB64, signature] = token.split('.');
    if (!headerB64 || !payloadB64 || !signature) return null;

    const expectedSig = crypto.createHmac('sha256', secret)
      .update(`${headerB64}.${payloadB64}`)
      .digest('base64url');

    if (signature !== expectedSig) return null;

    const payload = JSON.parse(Buffer.from(payloadB64, 'base64url').toString());
    if (payload.exp < Math.floor(Date.now() / 1000)) return null;

    return payload;
  } catch {
    return null;
  }
}
```

**Verify:** Sign a token, verify it — should round-trip correctly.

### Task 2.3: Create OAuth flow handler

**Objective:** Handle OAuth redirect, callback, token exchange

**Files:**
- Create: `src/auth/oauth-flow.ts`

```typescript
// src/auth/oauth-flow.ts
import { OAuthProvider } from './oauth-config.js';

export function getAuthRedirectUrl(provider: OAuthProvider, redirectUri: string, state: string): string {
  const params = new URLSearchParams({
    client_id: provider.clientId,
    redirect_uri: redirectUri,
    scope: provider.scope,
    response_type: 'code',
    state,
  });
  return `${provider.authUrl}?${params.toString()}`;
}

export async function exchangeCodeForToken(
  provider: OAuthProvider,
  code: string,
  redirectUri: string
): Promise<string | null> {
  try {
    const response = await fetch(provider.tokenUrl, {
      method: 'POST',
      headers: { 'Accept': 'application/json', 'Content-Type': 'application/json' },
      body: JSON.stringify({
        client_id: provider.clientId,
        client_secret: provider.clientSecret,
        code,
        redirect_uri: redirectUri,
      }),
    });
    const data = await response.json() as any;
    return data.access_token ?? null;
  } catch {
    return null;
  }
}

export async function getUserInfo(
  provider: OAuthProvider,
  accessToken: string
): Promise<{ id: string; email: string; name: string } | null> {
  try {
    const headers: Record<string, string> = {
      'Authorization': `Bearer ${accessToken}`,
      'Accept': 'application/json',
    };
    // GitHub needs User-Agent
    if (provider.id === 'github') headers['User-Agent'] = '8Router';

    const response = await fetch(provider.userInfoUrl, { headers });
    const data = await response.json() as any;

    if (provider.id === 'google') {
      return { id: data.id, email: data.email, name: data.name };
    }
    if (provider.id === 'github') {
      return { id: String(data.id), email: data.email ?? `${data.login}@github.com`, name: data.name ?? data.login };
    }
    return null;
  } catch {
    return null;
  }
}
```

## Feature 3: i18n (Internationalization)

**Why:** Support EN, ID (Indonesian), and JA (Japanese) for landing page and dashboard. Makes 8Router accessible to wider audience.

**Approach:** JSON string catalogs with a simple translation function. Landing page detects Accept-Language header. Dashboard has language switcher. No heavy i18n framework — just a `t(key, lang)` function.

```typescript
// src/i18n/index.ts
import en from './en.json' assert { type: 'json' };
import id from './id.json' assert { type: 'json' };
import ja from './ja.json' assert { type: 'json' };

export type Lang = 'en' | 'id' | 'ja';
export type TranslationKey = keyof typeof en;

const translations: Record<Lang, Record<string, string>> = { en, id, ja };

export function t(key: string, lang: Lang = 'en'): string {
  return translations[lang]?.[key] ?? translations.en[key] ?? key;
}

export function detectLang(acceptLanguage?: string): Lang {
  if (!acceptLanguage) return 'en';
  if (acceptLanguage.startsWith('id')) return 'id';
  if (acceptLanguage.startsWith('ja')) return 'ja';
  return 'en';
}

export function getSupportedLangs(): Lang[] {
  return ['en', 'id', 'ja'];
}
```
